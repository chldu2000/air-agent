# v0.8 Plugin Registry And Tool Ecosystem Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add local plugin discovery and registration through explicit `AgentConfig.plugins`, manifest authorization, and Python entrypoint functions.

**Architecture:** Plugins live in local directories with `air-agent-plugin.json`; the loader validates manifests, checks authorization, imports an entrypoint relative to the plugin directory, and passes a `PluginContext` for registration. Agent startup applies plugin outputs before final provider/planner setup while preserving default behavior when no plugins are configured.

**Tech Stack:** Python dataclasses, importlib, pathlib, existing Agent/ToolRegistry/SkillManager abstractions, pytest/pytest-asyncio, no new runtime dependencies.
