# v0.6 Plan-and-Execute Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an opt-in `plan_execute` strategy with a real MVP `LLMPlanner`, while preserving the default ReAct behavior.

**Architecture:** Introduce planner dataclasses/protocols in a focused module, then integrate plan execution as a separate Agent strategy. Plan steps execute sequentially through the existing ReAct loop without persisting transient step prompts, and plan/step lifecycle events reuse the existing tracing dispatcher.

**Tech Stack:** Python 3.11 dataclasses/protocols, existing `LLMProvider`, existing `RunEvent` tracing, pytest/pytest-asyncio, no new runtime dependencies.
